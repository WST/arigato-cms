Управление контентом сайта

Данная функция панели администратора является самой востребованной, так как с ее помощью и формируется весь контент (смысловая нагрузка) сайта.

Все разделы сайта отображаются в виде дерева. Можно добавлять новые разделы, редактировать уже существующие, удалять разделы, а так же перемещать их вверх или вниз по дереву. Разделы первого и второго уровня будут отображаться в двухуровневом меню сайта.

Добавление нового раздела заключается в заполнении набора полей предлагаемой формы:

Заголовок – краткий заголовок раздела сайта для отображения его в меню.

Название – полное название раздела.

Автор – имя автора, написавшего статью, содержащуюся в данном разделе. Параметр не является обязательным к заполнению.

Мнемоника – мнемонический идентификатор раздела (не содержащий русских и других букв национальных кодировок). Мнемоника используется для формирования адреса странички, указывать ее не обязательно. Если мнемоника не указана, то адрес странички будет содержать уникальный числовой номер раздела.

Ключевые слова – ключевые слова, характеризующие статью, размещенную в данном разделе сайта. Значение этого параметра будет подставляться в meta-тега keywords.

Иконка – иконка раздела. Необходимо указать путь, относительно папки images. Если данный параметр не указан, то раздел не будет иметь иконку.

Раздел – ссылка на родительский раздел в дереве разделов, то есть на тот раздел, в котором будет размещаться данный.

Дополнение 1 – указывает первый дополнительный блок, размещаемый на данном разделе помимо самой статьи. Список может меняться в зависимости от версии системы. Могут быть доступны следующие значения: список подразделов – позволяет вывести на странички сайта список всех дочерних разделов, что откроет доступ к разделам, находящемся на 3 и более нижних уровнях в дереве; ЧаВо – в разделе будет создан дополнительный блок часто задаваемых вопросов; фотоальбом – раздел будет содержать фотоальбом.

Дополнение 2 – указывает второй дополнительный блок, который, в отличии от первого, может содержать разбиение на страницы. Аналогично первому, список может отличаться в зависимости от версии системы. Доступны следующие значения: комментарии – в раздел будет добавлен блок комментариев пользователей; новости – данный раздел будет содержать новостную ленту; история обновлений – раздел будет содержать полную историю обновлений сайта, формирующуюся автоматически. Дополнение «История обновлений» актуально только для главной странички сайта.

Описание – краткое описание статьи (абстракт).

Текст – полный текст статьи.

Ниже указывается тип статьи:

Простой текст – статья оформляется в виде простого текста. Для разметки можно использовать BB-коды.

HTML – статья оформляется в виде текста с HTML-разметкой.

PHP – в статье можно использовать произвольный PHP-код для реализации каких-либо динамических элементов. Следующие три радио-переключателя указывают способ отображения данного раздела: ссылки на соседние разделы – создает на странички кнопки, ссылающиеся на предыдущую и следующую статьи сайта; открытие подраздела – позволяет автоматически открыть не текущий раздел, а первый его подраздел.

Вывод пути до раздела – если флажок установлен, то на страничке будет выведен полный путь по дереву разделов, начиная от корня сайта и до текущего раздела.

Отображать раздел в главном меню – если убрать этот флажок, то данный раздел не будет виден в меню сайта, однако доступ к нему сохранится.

Заметки администратора к разделу – дополнительное текстовое поле, позволяющее задать заметки администратора сайта к текущему разделу. Данные заметки будут отображаться при просмотре соответствующей странички сайта, но видеть их будет только администратор.

Если используется режим редактирования раздела, то смысл всех полей сохраняется. Однако они будут заполнены автоматически текущими значениями, благодаря чему можно будет изменить любые параметры раздела. В частности, для того, что бы переместить статью в другую ветку дерева разделов необходимо изменить значение параметра «Раздел».

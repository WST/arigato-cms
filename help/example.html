Полезные приемы при создании сайта

Рассмотрим некоторые полезные приемы, позволяющие реализовать на сайте, управляемом системой Arigato CMS, различные интересные элементы.

Журнал сайта

Под журналом сайта понимается страничка, содержащая в себе историю обновлений. При помощи такого журнала посетители сайта смогут быстро отыскать новые статьи, отследить новые комментарии к статья. Журнал может быть размещен на главной странице сайта либо в отдельном разделе. Для создания журнала требуется создать новый раздел сайта и указать в качестве второго дополнения «История обновлений». В истории обновлений будут перечислены все разделы сайта, указано время создания каждого из разделов, количество просмотров данного раздела, а так же количество комментариев пользователей. Кроме названия и ссылки на сам раздел указывается еще и название родительского раздела (идущего выше в дереве разделов). Новые разделы помечаются специальным знаком. Весь список может быть отсортирован по дате публикации либо по популярности.

Гостевая книга сайта

На большинстве сайтов в Интернете существуют гостевые книги – специальные странички, на которых все желающие посетители сайта оставляют свои отзывы, замечания и пожелания. Для реализации гостевой книги в системе Arigato CMS необходимо создать новый раздел и указать в качестве второго дополнения «Комментарии». Текст статьи оставляется пустым, что бы на страничке были только комментарии пользователей. В гостевой книге посетители могут указать свое имя, адрес электронной почты, адрес своего сайта и номер ICQ. Все эти параметры заносятся в Cookie пользователя, и при добавлении очередного отзыва в гостевой книге ему уже не придется заново набирать свои данные.

Письмо администратору сайта

Гостевая книга предполагает обратную связь, при которой все желающие будут видеть сообщения других пользователей. Если же посетитель захочет связаться лично с администратором сайта, то он может отправить ему письмо на электронный ящик. По умолчанию такая возможность поддерживается системой Arigato CMS, отключить эту функцию можно в панели администрирования, убрав в настройке блоков галочку «Письмо администратору сайта». Что бы отправить письмо администратору, необходимо перейти по специальной ссылке, которая видна на всех страничках сайта. Однако администратор может встроить ссылку на форму отправки письма в любую из статей. Ссылка должна иметь вид: <a href="index.php?action=send&amp;subj=Тема">...</a>.

Новости сайта

Для того, что бы делать объявления посетителям сайта идеально подойдет новостная лента. Для ее создания нужно добавить новый раздел и указать второе дополнение «Новости». Для добавления новостей необходимо авторизироваться, как администратор сайта, зайти в раздел с новостями и заполнить предлагаемую форму добавления новости. Последняя новость сайта будет видна на всех страницах в виде абстракта и ссылки на полную версию. Можно настроить систему так, что бы последняя новость выводилась только в течении некоторого время после ее публикации. Для этого в панели администрирования в разделе управления блоками сайта нужно задать параметр «последняя новость сайта» равным «есть в течении срока давности».

Фотоальбом

Для создания фотоальбома нужно добавить новый раздел и указать в качестве первого дополнения «Фотоальбом». Управление фотоальбомом во многом схоже с управлением новостями. Необходимо ввести пароль администратора, после чего зайти на страничку сайта с фотоальбомом. Появится форма, позволяющая добавлять новые фотографии. Сама фотография должна быть загружена на сервер в папку images. Необходимо указать имя файла с фотографией, название (короткое) и описание (более подробное) фотографии. Добавлять можно либо в начало фотоальбома, либо в конец. Но в любом случае порядок следования фотографий может быть изменен в дальнейшем. Фотографии будут выводиться в виде эскизов, которые получаются автоматически. Размеры эскизов и количество фотографий в одной строке можно задать в разделе настройки конфигурации панели администрирования.

Часто задаваемые вопросы

Если на сайте нужно разместить статью, содержащую некоторый набор вопросов и ответов на них, то при добавлении нового раздела надо указать в качестве первого дополнения «ЧаВо». Добавить новый вопрос и ответ на него можно тем же методом, что и добавить новость на сайте. То есть нужно ввести пароль администратора, зайти в раздел, предназначенный для сбора вопросов, и заполнить предлагаемую форму. В результате посетители сайта будут видеть список вопросов, щелкая по которым они смогут просмотреть ответы. Порядок следования вопросов может быть изменен администратором. Кроме того, в любое время можно будет исправить случайно допущенные ошибки, для чего служит кнопка «правка» возле каждого из вопросов.

Вывод списка статей

Если в каком-либо разделе сайта содержится некоторое количество подразделов, то можно вывести ниже текста статьи список этих подразделов. Для этого в требуемом разделе надо указать первое дополнение «Список подразделов». Все подразделы будут выводиться в виде ссылок с кратким поясняющим текстом (который берется из описания соответствующего подраздела). Учитывая тот факт, что автоматически создаваемое меню сайта только двухуровневое, то доступ к третьему уровню разделов посетители смогут получить только через список подразделов. Аналогично можно реализовать доступ и к четвертому уровню. Однако не стоит делать вложенность статей очень глубокой. Если все же глубина вложенности достаточно большая, то навигацию по сайту может облегчить указание параметра «Вывод пути до раздела» для всех статей, начиная с корневой. В этом случае вверху странички будет выводиться полный путь до текущей статьи по дереву разделов, что позволит переходить на любую из верхних статей за «один клик».

Ссылка из меню на внешние сайты

Что бы сделать ссылку на другой сайт прямо из главного меню нужно создать специальный раздел, содержащий редирект. Под редиректом понимается команда, заставляющая браузер пользователя принудительно перейти на другой адрес. Редирект может быть реализован на HTML (<meta http-equiv="refresh" content="1; url=адрес_сайта">), JavaScript (document.location="адрес_сайта";) или PHP (header ("location: адрес_сайта");). Для редиректа на HTML или JavaScript, нужно создать статью в HTML и записать в нее указанный выше код. Однако лучше использовать редирект на PHP, для этого нужно создать статью в формате PHP, а в качестве текста указать следующее:
<?php
header ("location: адрес_сайта");
?>
Где «адрес_сайта» – URL-адрес, куда будет выполнен переход. Если же ссылок на внешние сайты много, то проще их разместить не в меню, а в специальном блоке ссылок, управлять которым можно через панель администрирования.
